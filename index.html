Como funciona?

Por Dentro do Gerador de CPF

Um CPF válido não é apenas um número aleatório. Ele é composto por 9 dígitos base + 2 dígitos verificadores (DV). Nosso gerador cria os 9 dígitos base e, em seguida, calcula os DVs usando o algoritmo Módulo 11.

  1. Cálculo do 1º DV: Multiplicamos os 9 dígitos por uma sequência de pesos (10, 9, 8, ..., 2), somamos tudo e calculamos o resto da divisão por 11. Se o resto for 0 ou 1, o DV é 0. Caso contrário, o DV é 11 menos o resto.
  2. Cálculo do 2º DV: Repetimos o processo, mas desta vez usando os 9 dígitos base mais o 1º DV já calculado, e os pesos vão de 11 a 2.

Como funciona?

Por Dentro do Gerador de CNPJ

O CNPJ (12 dígitos base + 2 DVs) também usa o algoritmo Módulo 11, mas com uma sequência de pesos diferente.

  1. Cálculo do 1º DV: Multiplicamos os 12 dígitos base por uma sequência de pesos (5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2).
  2. Cálculo do 2º DV: O processo é repetido com os 12 dígitos base mais o 1º DV, usando uma nova sequência de pesos (6, 5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2).

Como funciona?

Por Dentro do Gerador de Cartão

Cartões de crédito válidos seguem o Algoritmo de Luhn (Módulo 10). É um sistema de verificação diferente do CPF/CNPJ.

  1. Prefixo (BIN): Os primeiros dígitos identificam a bandeira (ex: 4 para Visa, 51-55 para Mastercard).
  2. Dígito Verificador: O algoritmo percorre o número, dobrando valores e somando dígitos para validar o número final, garantindo que seja um múltiplo de 10.
